## Overview

Pepperstars integrates seamlessly with popular third-party tools to supercharge your workflows. You connect services like GitHub for version control, Slack for notifications, and Stripe for payments directly through our dashboard. These integrations reduce context switching and automate repetitive tasks, letting you focus on building innovative solutions.

## API Connections

Pepperstars provides a RESTful API for custom integrations. Authenticate requests with your API key and interact with endpoints for projects, users, and webhooks.

<Request tabs="JavaScript,cURL" show-lines="true">
  ```javascript
  const response = await fetch('https://api.pepperstars.dev/v1/projects', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${YOUR_API_KEY}`,
      'Content-Type': 'application/json'
    }
  });
  const projects = await response.json();
  ```
  ```bash
  curl -X GET https://api.pepperstars.dev/v1/projects \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json"
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "projects": [
      {
        "id": "proj_123",
        "name": "My App",
        "status": "active"
      }
    ],
    "total": 1
  }
  ```
</Response>

Use these parameters for the projects endpoint:

<ParamField path="id" param-type="string" required="false">
  Filter by specific project ID.
</ParamField>

<ParamField query="status" param-type="string" required="false">
  Filter projects by status: `active`, `archived`.
</ParamField>

## Setup Instructions

Follow these steps to integrate Pepperstars with GitHub.

<Steps>
  <Step title="Generate Token" icon="key">
    Create a personal access token in your GitHub settings with `repo` and `workflow` scopes.
  </Step>
  <Step title="Connect in Dashboard" icon="settings">
    Navigate to Integrations > GitHub in Pepperstars dashboard and paste your token.
  </Step>
  <Step title="Configure Webhook" icon="zap">
    Select repositories and events to monitor.
  </Step>
</Steps>

Install the Pepperstars CLI for local testing across languages.

<CodeGroup tabs="npm,yarn,pip">
  ```bash
  npm install -g @pepperstars/cli
  ```
  ```bash
  yarn global add @pepperstars/cli
  ```
  ```bash
  pip install pepperstars-cli
  ```
</CodeGroup>

## Platform-Specific Setup

<Tabs>
  <Tab title="Node.js" icon="nodejs">
    Initialize integration with this script:

    ```javascript
    const pepperstars = require('@pepperstars/sdk');
    const client = new pepperstars.Client({ apiKey: process.env.PEPPERSTARS_API_KEY });
    await client.integrations.connect('github', { token: 'ghp_YOUR_GITHUB_TOKEN' });
    ```
  </Tab>
  <Tab title="Python" icon="python">
    Set up using the Python SDK:

    ```python
    from pepperstars import Client
    client = Client(api_key="YOUR_API_KEY")
    client.integrations.connect("github", token="ghp_YOUR_GITHUB_TOKEN")
    ```
  </Tab>
</Tabs>

## Best Practices

<Callout kind="tip">
  Store API keys securely using environment variables, never hardcode them in your repository.
</Callout>

<Expandable title="Advanced Webhook Security" default-open="false">

Implement signature verification to ensure webhook authenticity.

```javascript
// Verify Pepperstars webhook signature
const crypto = require('crypto');
const signature = req.headers['x-pepperstars-signature'];
const hash = crypto.createHmac('sha256', YOUR_WEBHOOK_SECRET)
  .update(req.body)
  .digest('hex');
if (signature !== `sha256=${hash}`) {
  throw new Error('Invalid signature');
}
```

</Expandable>

<Callout kind="alert">
  Rotate integration tokens every 90 days to maintain security.
</Callout>

Follow these practices to maximize reliability and security in your Pepperstars integrations.
